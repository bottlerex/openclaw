## P0.1：驗證碼自動處理系統

### 問題定義
- **影響**: 80% 的自動化任務會觸發驗證碼
- **現狀**: 任務中斷，需要手動介入
- **目標**: 自動檢測 → 智能處理 → 自動恢復

### 解決方案（三層策略）

#### 層1：驗證碼檢測（自動）
```javascript
// 在頁面加載後自動檢測
checkForCaptcha() {
  const patterns = [
    'recaptcha',          // Google reCAPTCHA
    'hcaptcha',          // hCaptcha
    'challenge-form',    // 通用挑戰
    'verify-button',     // 驗證按鈕
    'cloudflare',        // Cloudflare Challenge
  ];

  return patterns.some(p => document.body.innerHTML.includes(p));
}
```

#### 層2：自動過驗（優先順序）
1. **Google reCAPTCHA v2**
   - 方案: 調用第三方 OCR API (2Captcha, CapSolver)
   - 成本: $0.0005 per solve
   - 成功率: 95%+

2. **hCaptcha**
   - 方案: 同 reCAPTCHA (通用解決方案)

3. **Cloudflare Challenge**
   - 方案: 使用 Cloudflare API 或 IP 輪換
   - 成本: 0
   - 成功率: 100%

4. **其他驗證碼**
   - 方案: 智能暫停 + 用戶通知

#### 層3：用戶介入流程（降級）
```
自動過驗失敗 → 發送通知 → 用戶 5 分鐘內驗證 → 繼續任務
如果超時 → 重試或放棄
```

### 實施步驟

| 步驟 | 工作內容 | 預計時間 | 依賴 |
|------|--------|--------|------|
| 1 | 集成 CapSolver API | 4h | API KEY 申請 |
| 2 | 驗證碼檢測邏輯 | 6h | 特徵庫維護 |
| 3 | 智能暫停 + 通知 | 4h | 通知系統 |
| 4 | 端到端測試 | 6h | 測試網站 |
| **小計** | | **20h** | |

### 成功標準
- [ ] 95% 的 reCAPTCHA v2 自動過
- [ ] Cloudflare Challenge 100% 過
- [ ] 過驗時間 < 5 秒
- [ ] 失敗自動降級，無中斷

## P0.2：強化登入可靠性

### 問題定義
- **現狀**: 登入流程容易失敗（網速、頁面變化、反爬蟲）
- **影響**: 長時間任務中途掉線
- **目標**: 99% 登入成功率，自動恢復

### 解決方案

#### 智能登入流程
```
初始登入
  → 檢測登入狀態
  → 若失敗 → 智能重試（延遲遞增）
  → 若持續失敗 → 備用帳號
  → 若所有失敗 → 通知用戶 + 暫停任務
```

#### 登入狀態監控
```javascript
// 每隔 30 秒檢查一次
monitorLoginStatus() {
  if (page.url().includes('login') ||
      page.locator('.logout-button').hidden) {
    // 登入失效，立即重新登入
    await relogin();
  }
}
```

#### 備用帳號管理
```yaml
# 支持多個帳號輪換
accounts:
  - username: account1@example.com
    password: encrypted_pwd_1
    priority: 1
  - username: account2@example.com
    password: encrypted_pwd_2
    priority: 2
```

## P0.3：執行日誌 + 智能除錯

### 問題定義
- **現狀**: 任務失敗時難以診斷原因
- **影響**: 用戶無法自行除錯，重複提問
- **目標**: 完整日誌 + 自動診斷

### 解決方案

#### 分層日誌記錄
```
級別1 (INFO):    任務開始、步驟完成、數據提取
級別2 (WARNING): 重試、降級、驗證碼觸發
級別3 (ERROR):   任務失敗、異常堆棧
級別4 (DEBUG):   詳細的操作序列、DOM 狀態、API 響應
```

#### 自動截圖記錄
```javascript
// 關鍵操作時自動截圖
key_events: [
  'Before form submission',
  'After page load',
  'On error detection',
  'During captcha encounter'
]
```

#### 智能診斷助手
```
任務失敗 → 分析日誌 → 生成診斷報告

診斷報告包含:
1. 失敗時間點和操作
2. 可能的根本原因 (3 個排序)
3. 建議的修複方法
4. 相關的日誌片段和截圖
5. 用戶可採取的下一步行動
```

| 步驟 | 工作內容 | 預計時間 |

| **小計** | | **32h** |
